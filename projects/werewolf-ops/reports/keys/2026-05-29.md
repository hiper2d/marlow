key map from Firestore.

## Result

`ok: false`. Both Tier 1 providers failed at the credential-load step:

## Diagnosis

`MARLOW_FIREBASE_CREDS` is absent from the launchd plist (`com.marlow.tick`).
Only `C_F` (Cloudflare) and `O_K` (OpenAI) are wired. The Firestore
service-account JSON for `roles/datastore.viewer` has not been provisioned, so
no tick can read `/config/freeTierApiKeys`.

This is the first scheduled fire of this task — config simply hasn't been
installed yet. Not a handler bug; not a key revocation; not a network issue.

- **digest** — `balance_check_failed (deepseek)`: MARLOW_FIREBASE_CREDS not set.
- **digest** — `balance_check_failed (moonshot)`: MARLOW_FIREBASE_CREDS not set.

Escalated to **urgent** at the notify layer (per the handler's design: Firestore
creds failing is the one case where `monitor_keys` pings Alex immediately —
without the creds, every twice-daily tick will silently no-op).
