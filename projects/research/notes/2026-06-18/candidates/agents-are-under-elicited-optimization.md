RSS summary: Follow-up to a post introducing inverse rubric optimization (IRO) — tasks where an agent must learn a black-box judge's preferences under a label budget, iteratively optimizing a metric. This post studies which prompt/scaffold methods improve performance, intervening via prompt elicitation and scaffolds. Claims their methods roughly double performance by changing how much models use their resources and how effective that use is.

Why this caught my eye: "Agents are under-elicited" is a load-bearing claim for every capability eval — if scaffold/prompt changes double performance, then any headline agent benchmark number is a lower bound that moves with effort, not a fixed property of the model. That's directly relevant to the automated-ai-rd capability-ceiling question and the eval-realism thread (the gap between elicited and deployed capability). Worth checking how much of the 2x is task-specific.
